## 摘要（Executive Summary）
本产品面向 **开发者** 与 **终端用户** 两类核心角色，提供“从 Dashboard 数据到可听解读”的闭环能力，解决 **数据可视化难以理解、协作共识难建立** 的问题。方案由三部分组成：  
1) **开发者 SDK**：注入多 panel 原始数据并在页面召唤演示模式 bar；  
2) **终端用户插件**：截图后调用多模态大模型生成文本解读；  
3) **语音合成服务**：将解读转为音频并播放。  
最终交付物是 **可解释、可传播、可演示** 的语音化洞察。

---

## 1. 背景与目标
### 1.1 背景
当下 Dashboard 的主要问题不是“看不到数据”，而是 **看得见但难以读懂**。跨角色沟通（研发、产品、销售、管理）时，理解成本与解释成本叠加，导致决策延迟与协作误差。

### 1.2 目标（Pyramid 结构）
- **顶层目标**：把复杂数据转化为可被听懂、可被复述的洞察叙事。  
- **支撑目标**：
  1) 让开发者以最低集成成本提供结构化语义数据；  
  2) 让终端用户无需接入源系统也能得到可靠解读；  
  3) 让解读结果以语音形态快速触达并可演示/复用。

---

## 2. 用户与场景（MECE）
### 2.1 用户角色
| 角色 | 目标 | 典型任务 | 成功标准 |
|---|---|---|---|
| 开发者（Developer） | 低成本接入 | 注册 panel 数据、配置快捷键与演示 bar | 集成时间 ≤ 1 天，运行稳定 |
| 终端用户（End User） | 快速理解并传播 | 截图、生成解读、播放语音 | 30 秒内完成一次可用解读 |
| 组织管理员（Admin） | 风控与成本控制 | 权限管理、语音与模型配置 | 合规、预算可控 |

### 2.2 关键使用场景
1) **产品/研发汇报**：开发者在演示时一键播放“全局解读”；  
2) **销售现场**：客户看不懂报表时，插件即时生成语音讲解；  
3) **远程协作**：在截图基础上生成可分享的音频与文字摘要。

---

## 3. 核心痛点与需求假设
### 3.1 痛点
1) Dashboard **缺乏语义层**：图表只是可视化，未提供故事线；  
2) **跨角色理解不一致**：同图不同解读，造成决策偏差；  
3) **演示效率低**：需要人为讲解且难以复用。

### 3.2 需求假设（可验证）
- 若系统能在 **≤5 秒** 输出解读语音，则用户愿意在会议或销售场景中持续使用。  
（注：该时间阈值为产品设计目标，非外部统计数据）

## 4. 产品范围与边界
### 4.1 In-Scope

## 6. 功能需求（FR）
### 6.1 SDK（开发者侧）
**FR-1 面板注册**  
**观点**：必须为每个 panel 提供可机器理解的语义结构。  
**论据**：无语义元数据会导致模型无法判断指标口径与比较维度。  
**要求**：  
1) 必填字段：`panelId`、`title`、`metricType`、`unit`、`data`、`timeRange`；  
2) 可选字段：`description`、`dimension`、`aggregation`、`thresholds`、`owner`。  

**FR-2 多 panel 数据注入与刷新**  
**观点**：同一页面可注册多个 panel，且支持增量更新。  
**要求**：  
1) 支持单 panel 更新（按 `panelId`）；  
2) 支持批量更新（数组提交）；  
3) 刷新间隔最小粒度 1 秒（防抖由 SDK 负责）。

**FR-3 快捷键召唤演示模式 bar**  
**观点**：演示场景需要无需鼠标的快速调用。  
**要求**：  
1) 默认快捷键：`Ctrl/⌘ + Shift + D`（可配置）；  
2) 支持快捷键冲突检测与提示；  
3) 支持隐藏/显示状态切换。

**FR-4 演示模式 bar 功能**  
**必须提供的操作入口**：  
1) 语音选择（至少 2 种风格，如“专业/友好”）；  
2) 播放全部（按 panel 顺序）；  
3) 播放指定 panel（下拉或列表选择）；  
4) 暂停/继续/停止；  
5) 播放速度（0.75x/1.0x/1.25x）。

**FR-5 播放反馈与状态**  
1) 显示当前播放 panel 名称与进度；  
2) 失败时提供错误原因与重试入口；  
3) 支持最小化状态悬浮。

**FR-6 事件与日志**  
1) 记录播放次数、失败率、播放时长；  
2) 支持导出或上报到客户自有埋点系统（可选）。

### 6.2 浏览器插件（终端用户侧）
**FR-7 截图采集**  
1) 支持区域截图与整页截图；  
2) 截图前需显示敏感信息提示（如“包含客户名称/金额”）；  
3) 支持局部涂抹/遮盖（首期可选）。

**FR-8 多模态识别与结构化解读**  
1) 输出结构需包含：`summary`、`highlights`、`risks`、`nextActions`；  
2) 识别失败时提供“低置信度提示”；  
3) 支持语言选择（至少中文/英文）。

**FR-9 语音合成与播放**  
1) 一键生成并播放音频；  
2) 可下载音频文件（mp3/wav）；  
3) 支持播放队列与停止。

**FR-10 结果管理**  
1) 最近 20 次解读可回看；  
2) 支持文本复制与分享链接（首期可选）。

### 6.3 后端与AI编排
**FR-11 多模态模型编排**  
1) 图像 → 结构化文本 → 叙事润色；  
2) 支持模型路由（不同模型处理不同任务）；  
3) 支持多租户隔离与配额控制。

**FR-12 语音合成与音频管理**  
1) TTS 生成时长与文本长度绑定（可限制）；  
2) 音频存储默认保留 7 天（可配置）；  
3) 支持自定义语音包（需授权）。

**FR-13 认证与权限**  
1) SDK 与插件均使用 API Token；  
2) Token 可撤销与轮换；  
3) 基于组织的访问控制（Admin 管理）。

## 7. 非功能需求（NFR）
### 7.1 性能
1) 文本解读生成 **P95 ≤ 3 秒**；  
2) 语音合成 **P95 ≤ 2 秒**；  
3) 端到端总耗时 **P95 ≤ 5 秒**。  
（均为产品设计目标）

### 7.2 可用性与稳定性
1) 服务可用性目标 **≥99.5%**；  
2) SDK 在单页上注册 50 个 panel 仍可稳定运行；  
3) 插件在 Chrome/Edge 最新 2 个大版本可用。

### 7.3 安全与隐私
1) 所有传输必须使用 TLS；  
2) 截图与原始数据默认 **不落盘**，仅保留处理缓存；  
3) 支持数据脱敏与删除请求（如企业客户要求）。

### 7.4 合规
若服务覆盖 EU/US 市场，需评估 GDPR/CCPA 适配（待法务确认）。

## 8. 数据与接口需求
### 8.1 Panel 数据结构（示例）
```json
{
  "panelId": "revenue_trend",
  "title": "月度收入趋势",
  "metricType": "time_series",
  "unit": "USD",
  "timeRange": "2025-01-01~2025-12-31",
  "data": [
    {"t": "2025-01", "v": 120000},
    {"t": "2025-02", "v": 135000}
  ],
  "dimension": "region",
  "aggregation": "sum",
  "thresholds": {"warning": 100000, "critical": 80000}
}
```

### 8.2 插件截图接口
输入：截图（PNG/JPEG） + 页面 URL + 语言偏好  
输出：结构化解读文本 + 语音文件 URL

### 8.3 SDK 与后端通信
1) `POST /panels/batch`：批量上报 panel 数据；  
2) `POST /narration/play`：请求生成语音；  
3) `GET /narration/status`：查询生成状态。

## 9. 运营与增长需求
1) 支持 **免费试用 + 付费订阅**（按组织）；  
2) 支持 **使用量计费**（按生成次数或音频时长）；  
3) 提供团队协作功能（共享语音包、模板）。

---

## 10. 风险与对策
| 风险 | 影响 | 对策 |
|---|---|---|
| 多模态模型误读图表 | 输出结论偏差 | 输出置信度提示 + 人工校验入口 |
| 截图包含敏感数据 | 合规风险 | 截图遮盖、脱敏、最小化保留 |
| 语音合成被滥用 | 品牌与法律风险 | 语音授权校验 + 水印标记 |
| 演示延迟过高 | 使用中断 | 缓存与并行处理优化 |

---

## 11. 成功指标（建议目标）
1) SDK 集成时间 **≤1 天**；  
2) 端到端解读完成时间 **P95 ≤ 5 秒**；  
3) 播放成功率 **≥98%**；  
4) 每个组织 7 日内 **≥3 次有效播放**；  
5) 终端用户满意度 **≥4.2/5**（主观评分）。

---

## 12. 里程碑与MVP范围
### 12.1 MVP（第一阶段）
1) SDK 支持 panel 注册 + 演示 bar；  
2) 插件支持截图 + 解读 + 语音播放；  
3) 单一语言（中文或英文）支持；  
4) 基础语音包（2 种）。

### 12.2 后续阶段（第二阶段）
1) 语音克隆与多语种；  
2) 自动摘要与分享链接；  
3) 企业级合规与审计。
